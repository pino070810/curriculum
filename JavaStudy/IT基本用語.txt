■ IT
IT...（Information Technology、情報技術） コンピュータとネットワークを利用した技術の総称
ICT...（Information and Communication Technology＝情報通信科学  コミュニケーション」という言葉が入っているため「個人からの情報発信」を強調するニュアンスがある
IoT...(Internet of Things  モノのインターネット) 従来インターネットに接続されていなかった様々なモノが、ネットワークに接続され相互に情報交換をする仕組み

■ OS
OS...オペレーティング・システム (Operating System) システム全体を管理し、さまざまなアプリケーションソフトを動かすための最も基本的なソフトウェア
Windows Microsoft社製OS。GUI
macOS (Mac OS X)...Apple社製OS。GUI
UNIX...米AT&T社のベル研究所製OS。近年はUNIXそのものが使われることは殆どなく、UNIXシステムをベースとしUNIX系OSとして利用されている。
Linux...Unix系OS。主にサーバー用として使われることが多い。

■ 言語・環境

マークアップ／フロントエンド
HTML...マークアップ言語。ページの要素や構造を指定するための言語。
CSS...スタイルシート。HTMLで作った要素を装飾するスタイルを指定する言語。
JavaScript...動的にコンテンツを更新、管理できるスクリプト言語。
jQuery...JavaScriptライブラリ。ウェブブラウザ用のJavaScriptコードをより容易に記述できるようにするために設計された。
Java SE...Java Platform, Standard Editionの略称。標準API、基礎となる標準的な機能をまとめたもの。

サーバーサイド
.NET（C#, VB）
Java...オブジェクト指向のプログラミング言語。プラットフォームに依存しない為、OSに関係なく動作させることができる。
Java SE...Java Platform, Standard Editionの略称。標準API、基礎となる標準的な機能をまとめたもの。
Java EE（Jakarta EE）...Java 2 Platform, Enterprise Editionの略称。前述のSEに、企業用途(Enterprise)のサーバー用の機能が追加されたAPI。
Perl...歴史あるスクリプト言語。テキスト処理が優れていが、ウェブ・アプリケーション、システム管理等にも用いられる。
PHP...サーバーサイドのスクリプト言語。HTMLやCSSを組み合わせ。動的にWebページを生成することができる。
Ruby...オブジェクト指向スクリプト言語。Ruby on RailsというWebアプリケーションフレームワークを使うことができる。

■ サーバーソフト
Apache...Apache HTTP Server（アパッチ エイチティーティーピー サーバ)。オープンソースのクロスプラットフォームのWebサーバソフトウェア。
nginx...nginx「エンジンエックス」。オープンソースなWebサーバで、処理性能・高い並行性・メモリ使用量の小ささに焦点を当てて開発されている。
IIS...Microsoft Internet Information Services (IIS) 。Microsoft Windowsの標準Webサーバー（アプリケーションサーバー）サービスである。

■ SQL
SQL...「Structured Query Language」の略。プログラミング言語ではなく、リレーショナルデータベース(RDB)のデータを操作するための言語。
DB2...DB2とは、IBM社のリレーショナルデータベース管理システム（RDBMS）の総称。UnixやLinux、Windowsなどの幅広いプラットフォームに対応。
MySQL...GNU General Public Licenseの条件に基づいたフリーでオープンソースのソフトウェア。多くのデータベース駆動型Webアプリケーションで使用されている。
Maria DB...MySQL派生として開発されているRDBMS。MySQLと非常に高い互換性を持っている。 
Oracle DB...Oracle社が開発・リリースしているRDBMS。Windows、Linux等に対応し複雑な処理も可能としている。
PostgreSQL...拡張性とSQL準拠を強調するRDBMS。組み込みアプリケーションから大規模データ・ウェアハウスまで幅広く対応。
SQLite...サーバとしてではなくアプリケーションに組み込んで利用されるデータベース。RDBMSと違い、APIは単純にライブラリを呼び出すだけであり、データの保存に単一のファイルのみを使用
SQL Server...正式名称「Microsoft SQL Server」。Microsoft社が開発したRDBMSでコマンドを入力して操作する「CLI（CUI）」ではなくデータベースを視覚的に操作する「GUI」が利用できる。
CRUD...Create（生成）、Read（読み取り）、Update（更新）、Delete（削除）。コンピュータソフトウェアが持つ永続性[1]の4つの基本機能のイニシャルを並べた用語。
トランザクション...結果の整合性が要求される、複数の処理を1つにまとめたもの。これら「複数の処理」は分離させることはできず、結果は成功か失敗のいずれかであること。
コミット...トランザクション（データベースにおける一連の更新処理）が全て正常に完結したという宣言のこと。
ロールバック...障害発生時に処理途上でコミットされていないトランザクションの処理をすべて取り消し、最初の時点に戻すことを意味する。
排他制御（排他ロック）...ダブルブッキングしないようにすること。排他ロック中は他人が見たり変更したりできなくなる。共有ロックは見ることはできるが変更はできない。
楽観ロック...ダブルブッキングしない前提でロックをせずにデータを取得時と更新時のバージョンを確認して整合性を求める手法。
悲観ロック...ダブルブッキングする前提で1人がデータを取得するとロックがかかり、他人は取得ができなくなることで整合性を求める手法。
実行計画...より短い時間でSQLを実行できる方法を計算し、具体的な方法を「計画」としてまとめたもの。
チューニング...システムやプログラムなどを改良し、パフォーマンスの向上を図ること
パフォーマンス...処理性能やネットワークなどの伝送速度・容量などのこと
スロークエリ...処理速度が遅いSQLのこと
クエリログ...クライアントからの MySQL Server への接続・接続解除の情報、およびクライアントから実行された全ての SQL クエリを出力してくれるログのこと。
IDE（統合開発環境）...「Integrated Development Environment」の略。ソフトウェアの統合的な開発環境のこと。
Eclipse...標準でJavaの開発環境が含まれている。便利なプラグインも豊富に用意され、さまざまな言語の開発環境として使用可能。
NetBeans...プロジェクト管理機能や入力支援機能を備えたオープンソースのIDE。Java EEなどJavaによるWebサービス開発のための環境も用意されている。
SpringToolSuite...Eclipseをベースとして作成された統合開発環境(IDE)。Spring Frameworkを利用した開発に適した機能が備えられている。
DBツール...データベース管理タスクに使用できるツール、ユーティリティおよび補助ツールの総称。
A5:SQL Mk-2...データベース開発において生産性を向上させるべく開発されたSQL開発ツール。SQLを実行したり、テーブルを編集するほかに、SQLの実行計画を取得したり、ER図を作成したりすることが出来る。
pgAdmin...PostgreSQLの管理ツールの一つ。ネットワークを通じて遠隔のデータベースサーバをGUIで管理することができる。
DBeaver...ほぼ全てのDBに対応したSQLクライアントツール。Windows版・MacOSX版・Linux版がある。
MySQL Workbench...データーベースの管理から設計・開発までフルで行え、GUIベースで操作できる。
Sequel Pro...Mac用のMySQLのデータをGUI操作で管理するためのアプリケーション。Windows専用は、HeidiSQLがある。
TeamSQL...2019年2月末サービス終了。Windows、Mac、Linuxに対応。DataRowに移行された。
■ 肩書
PM...「プロジェクトマネージャー」。企業内でのプロジェクトを総合的に任され、計画と実行の責任を担う役職。
PL...「プロジェクトリーダー」。PMの立てた計画に沿って、プロジェクト全体を率いていく。
PG（PE）...「プログラマー」SEが作成したシステムの設計書通りに、詳細設計を行いプログラミングする役職。
SE...「システムエンジニア」。「システムの設計」と「クライアントとのやり取り」を担当する役職。
デザイナー...クライアントや依頼者の求めに応じたデザインを考案する役職。いくつもの○○デザイナーと呼ばれる役職がある。
エンジニア...「工学（エンジニアリング）に関する専門的な知識やスキルを持った人材」を指す言葉。
フロントエンド・エンジニア...Webサイトの閲覧や操作などのユーザーが直接見たり、触れたりする部分（フロントエンド）を開発するエンジニア。
サーバーサイド・エンジニア...サーバー側で行う処理に対して必要なプログラムの開発や、サーバーで扱うデータの管理を行うエンジニア。
バックエンド・エンジニア...サーバサイド全般を取り扱うエンジニアのこと。「サーバ構築」と「サーバサイドプログラム開発」が主に行う。
■ UI（ユーザーインターフェース）
UI...ユーザーインターフェイス（User Interface）。ユーザー（利用者）と製品やサービスとのインターフェース（接点）すべてのこと。
CUI...キャラクタユーザインターフェース (Character-based User Interface)。キーボードなどを用いてすべてのやり取りを文字によって行う方式。
GUI...グラフィカルユーザーインターフェース(グラフィカルユーザーインターフェース)。出力にグラフィックス表示を多用し、操作をマウスなどで主に画面上の位置を指定して行う。
■ ファイル／ディレクトリ
ファイル...外部記憶装置を使って情報をひとつのかたまりとして扱うための単位
拡張子...ファイルの種類を識別するためにファイルの名前（ファイル名）の末尾につけられる文字列。
ディレクトリ...ハードディスク上でファイルが記録されているデータの保管場所。いわゆる「フォルダ」のこと。
ルート・ディレクトリ...階層型ファイル構造における「最上階層」のディレクトリのこと。
カレント・ディレクトリ...階層型ファイルシステムを使用している時の、「現在の位置」であるディレクトリ。
パス...ファイルやフォルダの置いてある場所を示す情報。
絶対パス...ルートディレクトリから目的地までの経路のこと。
相対パス...カレントディレクトリを基準にして、目的地までの経路のこと。
■ ネットワーク
LAN...ローカルエリアネットワーク(Local Area Network)。屋内など一定の限定されたエリアで接続できるネットワークのこと。
WAN...ワイドエリアネットワーク(Wide Area Network)。世界中で繋がるネットワークのこと。
クライアント...利用者側のコンピュータのこと。
サーバー...サービスを提供する側のコンピューターのこと。
■ 各種プロトコル
HTTP...データをサーバとクライアントの間で通信するときのプロトコル(きまりごと)
FTP...ファイルを送受信する時に使う決まりごと（通信プロトコル）。FTPをコンピューターが認識できないと、ファイルの送受信はできない。
HTTPS...通信内容が暗号化されたHTTPのこと
SSL...送受信しているデータを暗号化する通信手順
SSH...安全に通信を行って、ネットワークに接続された機器を遠隔操作するための通信手段（プロトコル）の１つ
ポート、ポート番号...コンピュータで、実際に通信される出入口の番号。URLやIPアドレスが住所とすると、ポート番号は複数ある玄関のようなもの
HTTP、FTP、SSHの各ポート番号...プロトコルにより特定のポート番号が割り振られており、これをウェルノウンポートという。HTTP: 80番 FTP: 21番 SSH: 22番 HTTPS: 443番
ファイアウォール...外部からの不正なアクセスを防ぐシステム
TCP...送ったデータが相手に届いたか、その都度確認しながら通信するやり方で、「信頼性は高いが転送速度が低いという特徴」をもつ
■ IPアドレス
IPアドレス...インターネット上に接続された機器が持つナンバーのこと
グローバルIPアドレス...インターネットに接続するタイプのIPアドレス
プライベートIPアドレス...インターネットに接続しないタイプのIPアドレスで、直接グローバルIPアドレスとやりとりすることができない
■ ドメイン
ドメイン...インターネット上の住所のようなもの。「https//」 のあとに続く「example.com」の部分がドメイン
DNS...ドメイン名とIPアドレスの対応付けや、 メールの宛先ホストを指示するためのシステム
VPN...インターネットなどに接続している利用者の間に仮想的なトンネルを構築し、プライベートなネットワークを拡張する技術
ブリッジ...ネットワーク中のMACアドレスを見て、行き先を振り分けてくれる機器
プロキシサーバー...内部と外部のネットワークの中継をする役割を持つサーバーのこと。コンピューターに代わってインターネットに接続しサイトへアクセスしてくれる。
■ 電子メール
SMTP...「Simple Mail Transfer Protocol（シンプル・メール・トランスファー・プロトコル）」メールを相手のメールサーバーまで届ける仕組みのこと。
POP...「Post Office Protocol（ポスト・オフィス・プロトコル）」メールソフトが自動的にメールサーバーにアクセスしてメールをダウンロードする仕組みのこと。サーバーにデータが残らない
IMAP...「Internet Message Access Protocol（インターネット・メッセージ・アクセス・プロトコル）」メールソフトが自動的にメールサーバーにアクセスしてメールをダウンロードする仕組みのこと。サーバーにデータが残る。
■ セッションとクッキー
セッション...HTTPにおけるWebサーバとWebブラウザ間で状態を管理するプロトコル。そこで用いられるWebブラウザに保存された情報のこと。
クッキー...ユーザーの情報をサーバーに一時的に格納する仕組み。Cookieがクライアント側の情報維持の仕組みであるのに対して、セッションはサーバ側の仕組み。
キャッシュ...読み出したデータのうち、直近に読み込んだものや使用頻度が高いものを高速な記憶装置に複製しておくこと。
■ DB（データベース）について
インデックス...データをすばやく検索・抽出するために作成された索引データのこと
■ テストについて
フロー...コンピュータプログラムを構成する命令や手続き、処理の流れなどのこと
■ チケット管理ツール
タスク...コンピューターが処理するプログラムの処理単位のこと。
■ バージョン管理
Git...「分散型バージョン管理」ローカル環境で履歴を管理できるが使用方法など覚えることが多い
SVN...「集中型バージョン管理」SVNを扱う際も操作がとてもシンプルだがバージョンの管理がローカルでできない
デグレ...「degrade」（デグレード）以前のバージョンよりもシステムの品質が低下するトラブルのこと
マージ...「統合する」「合体する」等の意味合い。バージョン管理システムなどで管理している開発中のプログラムなどに、新たにコードを追加したり、修正を反映させたりする操作のこと
マージツール...マージするツール。ものによってファイルの比較し色分けしたり2つのファイルを結合できる。
